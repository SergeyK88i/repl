Групповой чат процесса

Это демонстрационный интерфейс: сценарий запускается из чата, а ответы агентов появляются с паузами, чтобы было видно, кто с кем общается.

Схема взаимодействия агентов

До READY агенты готовят источник. После READY стартует контур создания и настройки реплики.

До READY: подготовка источника

Координатор принимает предзаказ, ведёт статус, делегирует работу
WARP проверяет готовность и возвращает failed criteria
CR Manager создаёт CR-задачу и выполняет исправления
Пользователь создаёт предзаказ → Координатор фиксирует `correlation_id`.
Координатор спрашивает WARP: готов ли источник?
Если NOT_READY, Координатор передаёт failed criteria CR Manager.
CR Manager исправляет и сообщает Координатору результат self-check.
Координатор делает официальный final-check через WARP.
✓
READY

После READY: создание реплики

ЕР-Конфигуратор создаёт заказ в другом Jira-пространстве и генерирует конфиг
Init Loader запускает первичную загрузку данных
Публикатор проверяет и публикует результат
Координатор передаёт READY-предзаказ ЕР-Конфигуратору.
ЕР-Конфигуратор подбирает параметры реплики.
ЕР-Конфигуратор генерирует конфиг для ЕР.
Init Loader запускает загрузку.
Публикатор завершает процесс и делает результат доступным.
